# n8n Pharmaceutical Supply Chain Workflow Guide

## Overview
This workflow generates pharmaceutical supply chain graphs from webhook input, validates the AI output, persists nodes and edges to the backend, computes risks, and returns a workspace ID for frontend visualization.

- Input: Product demand scenario JSON via webhook
- Output: Success response with workspace ID, counts, assumptions, warnings
- Total nodes in workflow design: 28

## Expected Input Payload
```json
{
  "product_name": "Insulin Glargine",
  "api_ingredient": "Insulin Glargine",
  "formulation_type": "Injectable",
  "target_market": "USA",
  "forecast_demand_units_month": 50000,
  "require_gmp": true,
  "require_fda": true,
  "cold_chain_required": true,
  "max_lead_time_days": 45,
  "budget_usd": 2000000,
  "supplier_candidates": []
}
```

## Stage 1: Input and AI Generation

### Node 1: Webhook Trigger
- Type: Webhook (POST)
- Purpose: Entry point for demand scenarios
- Common issues:
  - 0 items when no POST is sent
  - 404 when URL/method mismatch

### Node 2: Supply Chain Planner (AI)
- Type: AI Chat Model (Gemini/OpenAI)
- Model options: gemini-1.5-flash or gpt-4o
- Temperature: 0.2
- Max tokens: 2500
- Require Specific Output Format: Off
- Purpose: Generate supply chain JSON with nodes, edges, assumptions, warnings
- Common issues:
  - Quota exceeded
  - Model name mismatch
  - Output includes markdown/text and not pure JSON

### Node 3: Validate Graph Structure
- Type: Code node (JavaScript)
- Purpose:
  - Extract JSON from model output
  - Validate and repair nodes/edges
  - Enforce allowed node types and numeric ranges
  - Remove invalid edges
- Output keys:
  - nodes
  - edges
  - assumptions
  - warnings

## Stage 2: Validation Gate

### Node 4: Check Validation Result
- Type: IF
- Condition: nodes is not empty
- True: Continue
- False: Return validation error

### Node 5: Return Validation Error
- Type: Respond to Webhook
- Purpose: Return structured failure message

## Stage 3: Workspace Creation

### Node 6: Create Workspace
- Type: HTTP Request
- Method: POST
- URL: http://localhost:5000/api/v1/workspaces
- Body:
```json
{
  "name": "AI Generated Supply Chain",
  "description": "Generated by n8n workflow"
}
```

### Node 7: Prepare Workspace Payload
- Type: Code node
- Purpose: Attach workspace ID to each node/edge payload

### Node 8: Return Workspace Creation Error
- Type: Respond to Webhook
- Purpose: Error path for workspace creation failures

## Stage 4: Node Creation

### Node 9: Split Nodes for Creation
- Type: Split Out
- Purpose: One item per node

### Node 10: Create Node
- Type: HTTP Request (loop)
- Method: POST
- URL: http://localhost:5000/api/v1/nodes
- Purpose: Persist each node

### Node 11: Collect Node Results
- Type: Aggregate
- Purpose: Gather results of node creation requests

### Node 12: Check Node Creation Failures
- Type: IF
- Purpose: Decide whether to continue to edges or return node error

## Stage 5: Edge Creation

### Node 13: Generate Edges from Nodes
- Type: Code node
- Purpose: Pull validated edges and attach workspace ID

### Node 14: Check If Edges Exist
- Type: IF
- Purpose: Skip edge creation when graph has no edges

### Node 15: Split Edges for Creation
- Type: Split Out

### Node 16: Create Edge
- Type: HTTP Request (loop)
- Method: POST
- URL: http://localhost:5000/api/v1/edges

### Node 17-20: Edge Error Handling
- Collect successes
- Collect failures
- Merge results
- Return edge creation error when critical failures occur

## Stage 6: Finalization

### Node 21: Compute Risks
- Type: HTTP Request
- Method: POST
- URL pattern:
  - http://localhost:5000/api/v1/graph/compute-risks?workspace={workspace_id}

### Node 22: Build Final Response
- Type: Code node
- Purpose: Build final success payload

### Node 23: Return Supply Chain Graph
- Type: Respond to Webhook
- Purpose: Return final workflow output to caller

## Execution Flow
Webhook -> AI -> Validation -> IF valid?
- If no: Return validation error
- If yes: Create workspace -> Create nodes -> Create edges -> Compute risks -> Return success

## Key Configuration Values
- Backend base URL: http://localhost:5000/api/v1
- AI model: gemini-1.5-flash or gpt-4o
- Temperature: 0.2
- Max tokens: 2500
- Target node count: 8 to 18
- Required presence: Manufacturer, Distributor, Retailer
- Score field range: 0 to 100

## Common Failures and Fast Checks
- Webhook 404 in test URL:
  - Click Execute workflow first, then send one request
- Webhook-test not registered:
  - Use production URL when workflow is active
- Validation failing often:
  - Keep output format strict toggle off and parse in Code node
- Workspace/node/edge HTTP failures:
  - Verify backend is running at localhost:5000
  - Verify endpoint paths match backend routes
- Edge creation failing:
  - Ensure all referenced source/target node IDs were created
